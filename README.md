## 🚀 快速開始

### 第一步：取得免費 API Key

前往 [Google AI Studio](https://aistudio.google.com/apikey) → 點擊 **Create API key**

- ✅ 完全免費，每日最高 **1,440 次**請求
- ✅ 無需綁定信用卡
- Key 格式：`AQ.Ab8R…` 或 `AIzaSy…` 兩種都支援

### 第二步：開啟系統

點擊上方線上連結，或下載 `index.html` 用瀏覽器開啟。

### 第三步：設定 API Key

進入 **⚙️ 設定** → 貼上 Gemini API Key → 點擊**儲存**

### 第四步：開始批改

1. 點選 **📷 批改考卷** → 建立考卷（填入科目、年級、學年度）
2. 上傳學生考卷照片
3. 若一位學生有多頁：上傳完第一位學生的所有頁面後，點擊 **＋ 新學生分隔**，再上傳下一位學生的照片
4. 點擊 **🤖 開始 AI 批改**，系統逐位學生分析

## ☁️ 雲端同步設定（選填）

若需將批改記錄同步至 Google 試算表，請依以下步驟設定：

1. 開啟一份 Google 試算表 → **擴充功能** → **Apps Script**
2. 將 `remediation-backend.gs` 的內容全部貼上（取代原有內容）
3. 點擊**部署** → 新增部署作業 → 網頁應用程式
   - 以下列身分執行：**我（Me）**
   - 誰可以存取：**所有人（Anyone）**
4. 複製部署網址
5. 回到系統：**⚙️ 設定** → 貼上網址 → **🔌 測試連線**

後台會自動建立兩個工作表：
- **批改記錄** — 所有學生批改結果
- **考卷設定** — 考卷基本資料

## 🫀 健康預測 App（LifeSpan）

檔案：`health.html`（單一 HTML，瀏覽器直接開啟，或部署至 GitHub Pages）

一套個人健康管理與延壽輔助系統，功能包含：

| 功能 | 說明 |
|---|---|
| 📊 綜合健康評分 | 整合心血管、身體組成、活動、飲食、睡眠、菸酒、心理、代謝 8 大面向 |
| ⏳ 生理年齡預測 | 依生活型態因子估算「身體年齡」與實際年齡的落差 |
| 🕰️ 預期壽命推估 | 基於實證流行病學因子估算壽命區間與「可再爭取的年數」 |
| 🔬 慢性病風險 | 心血管疾病、第二型糖尿病、代謝症候群 相對風險分級 |
| 📈 趨勢追蹤 | 儲存紀錄點，追蹤體重、血壓、血糖、健康分、生理年齡變化 |
| ✅ 每日習慣 | 喝水、運動、蔬果、睡眠等每日打卡與連續天數 |
| 💡 延壽行動清單 | 依影響程度排序的個人化建議（附預估延壽年數） |
| 🤖 AI 健康顧問 | 選填：接 Gemini / OpenAI 取得更詳細個人化建議 |

**預測模型**整合 BMI、腰高比、血壓、靜止心率、血糖/血脂、運動量、睡眠、飲食品質、吸菸、飲酒、壓力與家族病史等因子，採用 Mifflin-St Jeor 基礎代謝公式與 FINDRISC 風格風險積分。

### 🔗 資料串接（匯入）

系統可自動解析並帶入外部健康資料，全程在瀏覽器端完成、資料不外傳：

| 來源 | 檔案格式 | 自動擷取 |
|---|---|---|
| 🏥 **健康存摺**（My Health Bank） | FHIR Bundle / JSON | 血壓、血糖、糖化血色素、總膽固醇、HDL、體重、身高、心率（依 LOINC 代碼對應） |
| ⌚ **穿戴裝置** | Apple 健康 `export.xml` / Google Fit JSON | 每日步數→每週運動量、靜止心率、睡眠時數 |
| 📎 **現有 App / 附件** | JSON / CSV | 依欄位名（含中英文）自動對應各項檢驗值 |

> **關於健保：** 台灣全民健保基於資安設計，未開放第三方 App 直接讀取個人資料。正確且合法的流程是由本人到「健保快易通／健康存摺」匯出自己的資料後於此匯入 — 系統不會、也無法繞過此步驟直連健保伺服器。

### ☁️ 雲端後台與自動化（`health-backend.gs`）

以 Google Apps Script + 試算表作為個人健康資料的雲端中心：

- **儲存**：每次評估的快照存入「健康紀錄」工作表；匯入的檢驗值存入「檢驗匯入」。
- **比對**：後台自動比較最近兩次紀錄，回傳各指標的變化與「改善/退步」判定。
- **管理**：跨裝置雲端同步（上傳全部 / 從雲端拉取合併）。
- **自動化**：執行一次 `setupWeeklyTrigger()` 後，系統每週一早上 8 點自動產生**健康週報**（含趨勢比較與久未量測提醒），並寄送 Email 到你的信箱、存檔於「健康週報」工作表。

設定：開啟 Google 試算表 → 擴充功能 → Apps Script → 貼上 `health-backend.gs` → 執行 `setupWeeklyTrigger()` 授權 → 部署為網頁應用程式（執行身分：我；存取：所有人）→ 複製網址貼入 App「設定 → 雲端後台」→ 測試連線。前端以 `text/plain` POST 避免 CORS 預檢，可穩定直連。

> ⚠️ **免責聲明**：所有預測皆為基於公開流行病學研究的統計估算，僅供衛教與自我管理參考，**不能取代醫療診斷**。如有健康疑慮，請諮詢合格醫師。

隱私：所有資料僅存於瀏覽器 `localStorage`；AI Key 僅存本機、直接呼叫 API，不經第三方伺服器。
